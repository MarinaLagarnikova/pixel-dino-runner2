# Pixel Dino Runner

## Задача

Тебе предстоит реализовать браузерную игру в жанре **бесконечный раннер**.

**Стек:** JavaScript, CSS, HTML. Библиотеки — по твоему выбору.

Детали задачи будут раскрываться поэтапно — по мере выполнения каждого этапа ты получишь следующую инструкцию.

> Форкни репозиторий и приступай к первому этапу.

### Инструкции по запуску

**Запуск**: `npm ci && npx serve -p 3000` → `http://localhost:3000/`

### Правила

- Код пишет только агент (Claude). Вручную не редактировать.
- Можно изменять промпты и просить агента переделать или исправить.
- Коммиты не переписывать (no amend/rebase).
- При переходе к новому этапу написать в общий чат: `"Приступил к этапу N"` + ссылка на коммит.
- При стопоре — написать в тред к своему сообщению `"Приступил к этапу N"` что именно мешает. Ведущий подключится.

### Рабочий процесс

Каждый этап проходит по одной схеме:

```
Архитектура → План → Ревью плана → Реализация → Ревью кода → Исправления → Коммит
```

Для каждой фазы используется своя роль агента. При смене роли — `/clear`.

#### Артефакты рабочего процесса

Агент не помнит предыдущий разговор после `/clear`. Чтобы не терять контекст — сохраняй результаты работы каждой роли в `.md` файлы и передавай их в промпт следующей роли.

Все такие файлы размещай в `/docs/process/`. Структуру и названия файлов определяешь сам — исходя из того, какой флоу ты выстраиваешь.

Примеры ролевых промптов находятся в `/docs/roles/`.

#### Порядок коммитов на каждом этапе

1. После реализации — baseline-коммит: `"Этап N: добавлена фича X"`
2. После ревью и исправлений — коммит: `"Этап N: исправление ..."`
3. После HITL-правок (если были) — коммит: `"Этап N: HITL фикс ..."`

Коммиты не переписывать (no amend/rebase) — это намеренно: по истории коммитов видно, что получилось без ревью, что после, и какой эффект дало участие человека.

### Обязательные артефакты

По завершении хакатона у каждого участника должны быть:

- Рабочий код с тестами
- Чеклист выполнения с прогрессом по этапам
- `CLAUDE.md` — точка входа для Claude Code (ссылается на `AGENTS.md`)
- `AGENTS.md` — инструкции и правила для AI-агентов
- `README.md` — описание проекта и инструкция по запуску
- `ARCHITECTURE.md` — актуальная архитектура проекта
- `NOTES.md` *(опционально)* — личные заметки: что узнал нового, что оказалось интересным, какие проблемы возникли при работе с агентом

---

### Требования к качеству

- Код соответствует выбранной архитектуре.
- Код следует лучшим практикам: использует подходящие паттерны, не содержит антипаттернов.
- Покрытие тестами ≥ 80%.
- Все настраиваемые параметры вынесены в константы.
- Нет закомментированного кода в финальном результате.
- `npm run validate` проходит перед каждым коммитом.
